# MAX-APP Rules

## Economy Mode

* Do not scan the entire repository.
* Read only files explicitly mentioned by the user.
* Never read more than 5 files without permission.
* Ask before opening additional files.

## Workflow

* First show a short plan.
* Then list files you want to read.
* Wait for approval before making changes.
* Prefer minimal patches.

## Commands

* Do not run tests unless requested.
* Do not run git commit.
* Do not run git push.
* Do not run composer update.
* Do not run npm install.
* Do not run database migrations without approval.

## PHP

* Run php -l only on modified files.
* Do not reformat unrelated code.
* Do not modify database schema unless requested.

## Output

* Keep responses concise.
* Show only relevant diff.
* Avoid repository-wide analysis.
